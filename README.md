# Бекенд и клиентская часть для синхронизации данных в приложении Lampa + авторизация

> **Форк оригинального проекта** [DragonSpirit/LampacTgSync](https://github.com/DragonSpirit/LampacTgSync)

## 🆕 Отличия от оригинала

- **Управление пользователями** через Telegram бота (создание, удаление, редактирование)
- **Белый список** Telegram ID

## 📋 Возможности

### Для пользователей
- `/start` — получить пароль и код синхронизации для Lampa
- Автоматическое создание аккаунта при первом обращении
- Отображение срока подписки

### Для администратора
- 🆕 Создание пользователей вручную
- 👥 Просмотр списка пользователей
- 🗑 Удаление пользователей
- ✏️ Редактирование пользователей (группа, срок, комментарий, права Adult/Admin)
- ➕ Добавление в белый список Telegram ID
- 🗑 Удаление из белого списка
- 📋 Просмотр белого списка

## 🚀 Установка

Скачиваем из вкладки releases бэкенд.
Копируем `.env.example` в `.env`
```
cp .env.example .env
```
Заполняем необходимые параметры
```
USE_TELEGRAM_BOT: включить/отключить использование Telegram-бота (по умолчанию: true)
TELEGRAM_BOT_TOKEN: токен доступа Telegram-бота (по умолчанию: 123, брать у @BotFather в телеграмме)
TELEGRAM_ADMIN_ID: Можно указать несколько через запятую
PORT: порт, на котором запускается приложение (по умолчанию: 8080)
DISABLE_CORS: включить/отключить CORS (по умолчанию: true)
USE_STATIC_SERVER: включить/отключить статический сервер (по умолчанию: false)
```
и запускаем приложение: `LampacTgSync`, можно завернуть в systemd сервис:
```
[Unit]
Description=LampacTgSync
After=syslog.target
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/LampacTgSync
ExecStart=/opt/LampacTgSync/LampacTgSync
RestartSec=10
Restart=always

[Install]
WantedBy=multi-user.target
```
### Traefik (опционально)
```
tls:
  certificates:
    - certFile: /etc/letsencrypt/live/domen.ru/fullchain.pem
      keyFile: /etc/letsencrypt/live/domen.ru/privkey.pem

http:
  routers:
    lampa-tg-sync:
      rule: "Host(`lampa-tg-sync.domen.ru`)"
      entryPoints:
        - websecure
      service: lampa-tg-sync-service
      tls: {}

  services:
    lampa-tg-sync-service:
      loadBalancer:
        servers:
          - url: "http://127.0.0.1:8080"
```

При первом запуске в папке с приложением создастся файл `data.db`, в котором будут хранится все связанные с пользователями данные, в случае переноса достаточно перекинуть этот файл в новое место. Параметр `USE_STATIC_SERVER` позволяет отвязать клиентский плагин от `wwwroot lampac'а`, в там случае путь до файла будет http://IP:PORT/tg.js. Если нет желания использовать статику, то просто кидаем в `wwwroot/plugins` и в `lampainit-invc.my.js` прописываем
```
Lampa.Utils.putScriptAsync(["{localhost}/plugins/tg.js"]);
```
Перед этим не забываем прописать параметры для подключения в файле `tg.js` в секции конфигурации, либо просто определив `window.botLink` и `window.syncUrl` в том же `lampainit-invc.my.js` В случае если у вас не закрыто reverse proxy и есть проблемы с `CORS`, то можно отключить выставив параметр `DISABLE_CORS` в `true` (на свой страх и риск) При желании можно также отвязатся от telegram бота и создавать новых юзеров дёрнув руками пустой POST запрос на http://IP:PORT/user, в ответе придёт сегенерированный токен, либо добавляя новую забись любым редактором sqlite баз данных.

```
curl -X POST http://localhost:8080/user -H "Content-Type: application/json" -d '{}'
```

Важно! Если уже имеются локальные сохранённые данные, то во избежании их потери стоит перед включением синхронизации жмакнуть "Первичная синхронизация". В любом случае автор не несёт ответственность за потерянные данные просмотров, увы.

Также важно, что синхронизация работает по принципу, кто последний тот и батя, никаких CRDT и прочих фишек для разрешения конфликтов не реализовано и скорее всего не будет реализовано.

## 🚀 Сборка

```
git clone https://github.com/shurakenas/LampacTgSync.git
cd LampacTgSync
# Скачай зависимости
go mod tidy

# Собери бинарник для Linux (твой сервер)
GOOS=linux GOARCH=amd64 go build -o LampacTgSync .
```

## 📝 Логика работы
### Пользователь не в белом списке
```
⛔ Доступ запрещён.
📌 Ваш Telegram ID: 123456789
Сообщите этот ID администратору.
```
### Пользователь в белом списке
```
🔑 Ваш пароль для входа: 123456789
📅 Ваша подписка активна до: 29.04.2026
🔑 Ваш код для синхронизации: aB3dE5gH
```
### Администратор
```
👋 Администратор
📌 Ваш Telegram ID: 12345678
🔑 Ваш пароль для входа: 123456789
🔑 Ваш код для синхронизации: aB3dE5gH
Введите в настройках аккаунта Lampa.
```

## 🔑 Что такое пароль и код синхронизации
| Что | Значение | Где используется |
| --- | --- | --- |
| Пароль для входа | = Telegram ID пользователя (число) | Вводится при входе Lampa (если Lampac закрыт авторизацией) |
| Код синхронизации | 8-значный код (буквы и цифры) | Вводится в поле "Токен" в настройках аккаунта Lampa |


## ⚠️ Замечания

- Белый список — только пользователи из `whitelist.txt` могут взаимодействовать с ботом. Администраторы всегда в белом списке автоматически.
- Пароль — это Telegram ID пользователя (число). Для пользователей без Telegram пароль = ID, который указал админ при создании.
- При удалении пользователя запись удаляется из `/home/lampac/users.json` и из `data.db` (код синхронизации становится недействительным).
- Синхронизация работает по принципу "кто последний тот и папа"
- Перед синхронизацией нажми "Первичная синхронизация", чтобы не потерять данные
- Таймкоды синхронизируются при следующем открытии фильма
